    public interface UiWidget
extends EventSource<UiEvent,​UiEventListener>, AttributeReadValid, AttributeWriteVisible, AttributeWriteId
    Interface for an adapter to a physical widget of the underlying native widget toolkit. Anything displayed on the UI
    A widget is any object of the UI (user interface) and may be atomic or composite.
    Native widgets can be created. For convenience native widgets offer static of methods in their interface as a "constructor". So e.g. to create a new label simply use UiLabel#of(String) UiLabel.of("My Labeltext").
    If you want to make your UI code portable even for toolkits such as SWT, you need to make proper use of dispose() for all UiWidgets that are no longer needed what is generally a good idea. Further for toolkits such as the web with HTML/CSS/JS (via TeaVM) there are some limitations for the client-side code. Check out the documentation of TeaVM for further details.
    ATTENTION:
    A UiWidget can only be used once in a client application. Even if you want to have the exact same button twice on the screen you need to create two distinct instances.

    For advanced usage see also UiCustomWidget.

      • setReadOnly

        void setReadOnly​(boolean readOnly)
        Switches this widget between view (read-only true) and edit (read-only false) mode. This may also have effects such that a UiButtonPanel is showing a "Save" button only in edit mode, while it shows a "Edit" button in view mode that switches to edit mode.
        A UiComposite will propagate the read-only state to all its children when this method is called. Please note that several widgets such as UiLabel are always read-only. In such case this method will have no effect.
        Parameters:
        readOnly - the new value of isReadOnly().

      • setReadOnlyFixed

        void setReadOnlyFixed​(Boolean readOnlyFixed)
        Sets a fixed read-only mode. If not null the read-only mode will be set to that value and further invocations of setReadOnly(boolean) have no effect unless this method is called again with a value of null. Use this method for a widget (e.g. a UiInput) that should always be read-only. Unlike setReadOnly(boolean) this method is not recursive and will not propagate to child widgets.
        Parameters:
        readOnlyFixed - the new value of getReadOnlyFixed(). May be null to reset.
