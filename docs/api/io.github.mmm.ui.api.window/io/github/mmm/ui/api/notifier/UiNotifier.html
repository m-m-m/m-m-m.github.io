      • showPopupYesNo

        default UiPopup showPopupYesNo​(String message,
                               String title,
                               Consumer<Boolean> callback)
        This method opens a question popup window with the given message and two buttons for the options yes and no.
        ATTENTION:
        This is a very common also also very often misused feature of UI toolkits or frameworks. A suitable example is e.g. to use this with the message "Do you really want to delete this object?". However, it would still be more explicit to have the buttons labeled with "Delete" and "Cancel" using showPopup(String, UiSeverity, String, String, UiAction...). An example for a misuse of this method would be the message "Do you want to delete this occurrence or the series of the appointment? Press 'Yes' to delete the series and 'No' for this occurrence.". From this bad example that confuses the user, you should learn that it is always better to use more explicit button labels.
        Parameters:
        message - is the message to display.
        title - is the title that will be displayed in the title-bar of the popup. May be null to use the default title for UiSeverity.QUESTION.
        callback - is the Consumer invoked after the popup has been confirmed. Consumer.accept(Object) will receive Boolean.TRUE if the user clicked "Yes" and Boolean.FALSE for "No".
        Returns:
        the UiPopup.

      • showPopup

        default UiPopup showPopup​(String message,
                          UiSeverity severity,
                          String title,
                          String details,
                          UiAction... actions)
        This method opens a popup window with the given message. It is the most generic and flexible but also the most inconvenient showPopup method variant.
        Parameters:
        message - is the message to display.
        severity - is the UiSeverity.
        title - is the title that will be displayed in the title-bar of the popup. May be null to use the default title for the given UiSeverity.
        callback - is the Consumer invoked after the popup has been confirmed (the user has clicked on one of the buttons). The String value received by Consumer.accept(Object) will be the ID of the button that has been clicked what is the key of the given id2buttonLabelMap.
        details - the optional details. If not null details will be shown but are initially collapsed and have to be expanded by the end user.
        actions - are the UiActions for the buttons to close (answer, confirm, or cancel) the popup. Has to be at least one UiAction.
        Returns:
        the UiPopup.
