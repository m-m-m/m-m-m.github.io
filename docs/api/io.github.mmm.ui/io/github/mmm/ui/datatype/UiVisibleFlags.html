      • READ_ONLY

        static final InverseBitMask READ_ONLY
        InverseBitMask reserved for read-only (e.g. to hide mutable buttons like "Save").

      • AUTHORIZATION

        static final InverseBitMask AUTHORIZATION
        InverseBitMask reserved for authorization. Applications typically hide widgets (especially buttons) if the user does not have a general permission to invoke the underlying action. In such cases this BitMask shall be used. Please note for security the backend service needs to also perform the authorization and fail if not permitted. The hiding or disabling of widgets is only for usability.
