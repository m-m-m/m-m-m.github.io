      • setValue

        void setValue​(V value)
        Set both the value and the original value to the given argument. Further, modified is reset to false. In case the value type is mutable and you want to set the value to a clone or copy of the original value, then first call this method with the clone/copy and after that call setOriginalValue(Object) with the original value that you have cloned.

      • setValueForUser

        void setValueForUser​(V value)
        Like setValue(Object) but will not change the original value and will set modified to true (instead of false).
        This method is useful when you want to build a custom widget based on a UiInput that is read-only and does not allow the end-user to manually edit its value but via some other interaction such as a picker icon, a geographic map, or something alike.
        Parameters:
        value - the new value.

      • getOriginalValue

        V getOriginalValue()
        Returns:
        the original value since it was last set via setOriginalValue(Object) or most probably indirectly via setValue(Object). If this widget is modified, UiValuedWidget.getValue() will return the current value as entered by the end-user while this method will still return the original value.
        ATTENTION: The original value shall never be modified to avoid undesired side effects. Various features such as resetValue() rely on this fact.
        See Also:
        resetValue()
