    public interface UiWidget
extends EventSource<UiEvent,​UiEventListener>
    Interface for an adapter to a physical widget of the underlying native widget toolkit. Anything displayed on the UI
    A widget is any object of the UI (user interface) and may be atomic or composite.
    Native widgets can be created via UiContext that has implementations for all supported toolkits (technically via UiWidgetFactoryNative). This way you can decide which native toolkit you like to use just by configuration and dependencies.
    If you want to make your UI code portable even for toolkits such as SWT, you need to make proper use of dispose() for all UiWidgets that are no longer needed what is generally a good idea. Further for toolkits such as web (via TeaVM) there are strict limitations for the client-side code.
    ATTENTION:
    A UiWidget can only be used once in a client application. Even if you want to have the exact same button twice on the screen you need to create two distinct instances.

    For advanced usage see also UiCustomWidget.

      • getId

        String getId()
        This method gets the unique identifier of this object.
        ATTENTION:
        In order to be compliant with all possible UI toolkit implementations, a valid ID has to fulfill ECMA-262, Section 7.6. It is recommended to use IDs of the form [a-zA-Z][a-zA-Z0-9_$]*. Invalid IDs (e.g. containing colon, hash or period) are not accepted to prevent you from later having problems with CSS selectors.
        Returns:
        the ID of this object.
        See Also:
        ID_SEPARATOR

      • setReadOnly

        void setReadOnly​(boolean readOnly)
        Switches this widget between view (read-only true) and edit (read-only false) mode. This may also have effects such that a UiButtonPanel is showing a "Save" button only in edit mode, while it shows a "Edit" button in view mode that switches to edit mode.
        A UiComposite will propagate the read-only state to all its children when this method is called. Please note that several widgets such as UiLabel are always read-only. In such case this method will have no effect.
        Parameters:
        readOnly - the new value of isReadOnly().

      • getReadOnlyFixed

        Boolean getReadOnlyFixed()
        Returns:
        the fixed read-only mode or null if not fixed.

      • setReadOnlyFixed

        void setReadOnlyFixed​(Boolean readOnlyFixed)
        Sets a fixed read-only mode. If not null the read-only mode will be set to that value and further invocations of setReadOnly(boolean) have no effect unless this method is called again with a value of null. Use this method for a widget (e.g. a UiInput) that should always be read-only. Unlike setReadOnly(boolean) this method is not recursive and will not propagate to child widgets.
        Parameters:
        readOnlyFixed - the new value of getReadOnlyFixed(). May be null to reset.

      • setFocused

        default boolean setFocused()
        This method sets the focus to this widget. In case of a UiComposite it will set the focus to the first child capable of taking the focus.
        NOTE:
        You can only set the focus. To actually remove it, you need to set it in a different widget.
        Returns:
        true if the focus has been set successfully, false otherwise. You can normally ignore the result. It is only relevant for composite widgets such as panels, that delegate the call recursively and may not contain any child that can take the focus.

      • validate

        default boolean validate()
        Triggers a validation of this widget (including all potential children and propagation to parents). This will update the valid state and failure message of all involved widgets. For an immutable widget (e.g. a UiLabel) this method will have no effect and immediately returns true.
        Returns:
        the new valid status as result of this validation. In other words true if this widget and all its potential children have been successfully validated, false otherwise.
        See Also:
        UiValidatableWidget, isValid(), validateDown(UiValidState), validateUp(boolean)

      • validateUp

        default void validateUp​(boolean valid)
        Propagates the new validation state to parent (and recursively to ancestors). This allows UiTabs and other children from UiSingleComposites to indicate validation failures (e.g. with an error icon) so the end-user is able to find hidden children (e.g. inside tabs) and can correct the input data.
        Parameters:
        valid - the new valid status.
        See Also:
        validate()

      • isModified

        default boolean isModified()
        Returns:
        true if the value has been modified by the end-user via the UI since it has been set programmatically, false otherwise. A modified widget is also called dirty. A UiComposite is modified if one of its children are modified. An immutable widget (e.g. a UiLabel) will always return false here.
